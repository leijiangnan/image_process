# AGENTS.md

## 目的

这个仓库用于把九宫格或十二宫格源图处理成以下产物：

- 正方形 PNG 切图
- 一组左右轻微晃动的 GIF
- 一组人物缩小放大的 GIF
- 一组人物不动、文字单独缩小放大的 GIF

当用户说“处理图片”或直接提供图片路径时，默认按下面的流程执行。

## 默认图片处理流程

1. 将用户提供的本地图片路径作为输入源。
2. 每次处理优先创建新的输出目录，避免覆盖之前的结果。
3. 输出目录优先按图片时间戳命名，例如：
   - `output_20260426_154750`
   - 如果用户明确要求强制网格尺寸，可以按需要追加后缀，例如 `output_20260425_125121_4x3`
4. 在仓库根目录执行 CLI：

```bash
python3 -m image_grid_slicer.cli "/absolute/path/to/source.png" -o output_xxx
```

5. 如果用户明确指定了网格，必须始终传入 `--grid`。
6. 如果用户说明图片是 `4x3`，执行：

```bash
python3 -m image_grid_slicer.cli "/absolute/path/to/source.png" -o output_xxx --grid 4x3
```

7. 如果用户说明图片是 `3x3`，执行：

```bash
python3 -m image_grid_slicer.cli "/absolute/path/to/source.png" -o output_xxx --grid 3x3
```

8. 如果用户没有指定网格，就让程序自动识别。
9. 处理完成后，检查输出目录内容和 `manifest.json`。
10. 回复时要直接给出以下结果链接：
    - `slices/`
    - `gifs/`
    - `zoom_gifs/`
    - `text_blink_gifs/`
    - `manifest.json`

## 网格处理规则

- 如果源图有明显的黑色分割线，除非用户明确指定网格，否则优先使用自动识别。
- 如果源图没有黑色分割线，而用户已经明确知道布局，就优先使用用户给定的网格，而不是自动识别。
- 如果之前自动识别结果错误，而用户给出了正确网格，就用显式网格重新跑一次，并写入新的输出目录。

## 输出要求

每次成功处理后，通常都应该生成：

- `slices/`：正方形 PNG 切图
- `gifs/`：左右轻微晃动 GIF
- `zoom_gifs/`：人物缩小放大 GIF
- `text_blink_gifs/`：人物不动、文字或贴纸缩小放大 GIF
- `manifest.json`：所有输出文件的元数据

## 图片处理后的回复格式

每次图片处理完成后：

1. 明确说明图片已经处理完成。
2. 说明识别到的网格，或用户强制指定的网格。
3. 说明生成了多少张切图。
4. 给出输出目录。
5. 给出生成目录和 `manifest` 的可点击链接。
6. 如果有帮助，可以附上实际执行的命令。

## Git 与输出目录约定

- 除非用户明确要求，否则不要提交生成的 `output_*` 目录。
- 生成的输出文件默认只保留在本地供查看。
- 如果在改进流程时修改了代码，只提交源码和文档改动，除非用户明确要求提交产物。

## 排查说明

- 带黑线分割的图片通常裁切更准，因为程序可以检测到真实的网格边界。
- 没有分割线的图片，可能需要显式指定 `--grid 4x3` 或 `--grid 3x3`。
- 如果用户说上一版结果不对，优先按用户给定的正确网格重新执行，不要第一时间改代码。
